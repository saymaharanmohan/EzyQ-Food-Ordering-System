EzyQ Food Ordering System (EzyQ-FOS)

📌 Project Overview

EzyQ-FOS is a responsive web-based application developed for the Polytechnic Sultan Mizan Zainal Abidin (PSMZA) cafeteria. 

Developed during the COVID-19 pandemic, the system digitalizes traditional food ordering to comply with Ministry of Health (MOH) Standard Operating Procedures (SOP). 

It optimizes cafeteria operations by reducing peak-hour congestion and minimizing physical contact through a "Digital Queue" and "Takeaway" workflow.

🛠 Features & User Scopes

🔹 Administrator Module

User Management: Authority to register, view, update, and delete staff and customer accounts.

Menu Oversight: Full control over menu categories and items.

Business Intelligence: Access to weekly sales reports and system maintenance tools.

🔹 Cafeteria Staff Module

Order Management: Receive and process student orders in real-time.

Status Tracking: Update live order progress: Order Received → Preparing → Ready for take away.

Inventory Control: Add, edit, and delete daily menu items and categories.

🔹 Customer (Student) Module

Secure Portal: Registration and login via Matrix Number and encrypted password.

Digital Menu: Browse available food and beverages with real-time pricing.

Personal Dashboard: Monitor order status through four stages: Waiting, Preparing, Ready, and Finish.

Account Recovery: Built-in "Forgot Password" system with email reset link functionality.

📐 System Architecture & SDLC

The project followed the Waterfall Methodology over a 67-day development cycle (Sept 2021 – Dec 2021).

🔹 Logic & Documentation

Logical Design: Comprehensive implementation of Context Diagrams (CD), Data Flow Diagrams (DFD), and Entity Relationship Diagrams (ERD).

Physical Design: Responsive UI developed for Google Chrome and Microsoft Edge using Bootstrap principles.

Security: Features multi-level user authentication, encrypted passwords, and secure connection protocols.

🔹 Quality Assurance (QA)

Unit Testing (UTP): Successfully verified all modules for Admin, Staff, and Customer functions.

Integration Testing (ITP): Validated the interplay between the PHP application logic and the MySQL database.

User Acceptance Test (UAT): Validated by end-users (Aiman, Laila, and Firdaus) to ensure 100% functional alignment and ease of use.

📊 Benchmarking & Analysis

The system design was informed by a literature review of leading industry platforms, benchmarking against:

Belo Café: Responsive layout and menu navigation.

Lola’s Café: Interface organization and visual appeal.

Domino’s Pizza: Online ordering workflow and takeaway efficiency.

⚠️ System Limitations

Record Management: Previous order history currently cannot be deleted from the student dashboard.

Cancellation Policy: Students cannot cancel an order once the transaction has been submitted through the system.
